Web Part connections=Good

I recently finished up a project over at symantec. Part of their requirements was a sort of 3 tiered project creation dashboard. They wanted the ability to group tasks and documents to a specific list. Enter web part connections. What I ended up doing was adding a lookup field to the task list and document library, that referred back to the create a task custom list. Once I did that i was able to create a blank web part page and add the 3 lists as web parts. I then we connected them with the lookup field using the web part connections option. Now I am able to select a specific project and all the tasks and documents associated with it. Pretty nifty eh?
